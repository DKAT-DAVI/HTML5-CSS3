Listas em HTML

Ordered Lists

Todas as listas onde a ordem dos intens é algo muito importante: Um passo-a-passo para fazer um bolo, uma lista de aprovados, uma lista de de chamada...

Para criar uma ordered list, usamos a tag <ol> (ordered list) e para delimitar a lista <li> (list item) para identificar cada item da lista.

A partir do HTML5 a tag <li> tem fechamento </li> opcional

  1. Item 1
  2. Item 2
  3. Item 3
  4. Item 4
  5. Item 5

A tag <ol> possui o parâmetro type para definir o tipo de marcador da lista:

Você também pode indicar com qual número ou parâmetro irá iniciar a contagem usando o parâmetro start

  1. 1 - Valor padrão. Cria listas numeradas. Ex: 1, 2, 3, 4, …
  2. A - Cria listas alfabéticas em maiúsculas. Ex: A, B, C, D, …
  3. a - Cria listas alfabéticas em minúsculas. Ex: a, b, c, d, …
  4. I - Cria listas com algarismos romanos em maiúsculas. Ex: I, II, III, IV, …
  5. i - Cria listas com algarismos romanos em minúsculas. Ex: i, ii, iii, iv, …

Unordered lists

Também chamadas de listas com marcadores, aquelas que a ordem dos item não influencia no significado da lista. Maneira para organizar itens que não apresentam uma classificação necessáriamente.

Para criar uma unordered lists usamos a tag <ul> (unordered lists) e a tag <li> para criar cada um dos seus itens internos.

  • Item 1
  • Item 2
  • Item 3
  • Item 4
  • Item 5

O marcador padrão é o circle, a bolinha preta totalmente preenchida, mas existe o parâmetro type com os seguintes valores:

  • disc - padrão. Uma bola preta totalmente pintada
  • circle - Uma bola com uma borda preta e sem preenchimento
  • square - Um pequeno quadrado preto totalmente pintado

Usando links externos

Para criar um hyperlink, devemos criar âncoras através da tag <a>. O principal atributo dessa tag é o href, que cria uma referência hipertexto.

Usando a tag de hipertexto:

Meu perfil do GitHub

Dentro do atributo href, colocamos a URL para outro site.

Outro atributo bem útil da tag de âncora é o hreflang, que permite indicar qual é o idioma principal do site para onde o link está desviando o fluxo de navegação. Isso vai permitir avisar ao navegador e a softwares de tradução como lidar caso o visitante opte por traduzir automaticamente os conteúdos.

Site da W3Schools (em Inglês)

Podemos controlar como o link irá abrir, usando o atributo target com os seguintes valores:

  • _blank vai abrir o link em uma nova janela em branco. Meu perfil do GitHub.
  • _self vai abrir o link na janela ou frame atual (padrão).Meu perfil do GitHub.
  • _top vai desfazer todos os frames e abrir o destino no navegador completo. Meu perfil do GitHub.
  • _parent similar ao uso do _top em uma referência à janela mãe.Meu perfil do GitHub.
  • nome-do-frame caso esteja usando frames, indicar o nome da janela a abrir.

Existe um recurso bem interessante para links que é indicar qual é a natureza do destino usando o atributo rel. Esse atributo aceita vários valores, entre eles vou citar:

  • next indica que o link é para a próxima parte do documento atual.
  • prev indica que o link é para a parte anterior do documento atual.
  • author indica que é um link para o site do autor do artigo atual.
  • external indica que é um link para outro site que não faz parte do site.

Existe um recurso para para indicar a origem do link, utilizando a tag <rel>. Essa tag aceita os seguintes atributos.

  • next indica que o link é para a próxima parte do documento atual.
  • prev indica que o link é para a parte anterior do documento atual.
  • author indica que é um link para o site do autor do artigo atual.
  • external indica que é um link para outro site que não faz parte do site atual.
  • nofollow indica que é um link para um site não endossado, como um link pago.
